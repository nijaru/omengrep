### Task 1: Bump Version

**Files:**

- Modify: `Cargo.toml` (version field)

**Step 1: Edit version**

In `Cargo.toml`, change:

```toml
version = "0.0.1"
```

to:

```toml
version = "0.0.2"
```

**Step 2: Update lockfile**

```bash
cargo build --release 2>&1 | tail -3
```

Expected: `Compiling omengrep v0.0.2 ... Finished`

**Step 3: Run tests**

```bash
cargo test 2>&1 | tail -5
```

Expected: `14 passed; 0 failed`

**Step 4: Commit**

```bash
git add Cargo.toml Cargo.lock
git commit -m "Bump version to 0.0.2"
```

---

### Task 2: Write CHANGELOG

**Files:**

- Create: `CHANGELOG.md`

**Step 1: Create file**

```markdown
# Changelog

## [0.0.2] - 2026-03-03

### Fixed

- **Scope filter bug**: Searching `./src/cli` no longer returns results from
  `src/cli_utils/` — path prefix check now requires exact match or trailing `/`
- **Markdown chunk IDs**: Long markdown sections split into multiple chunks no
  longer silently drop all but the last chunk (duplicate ID overwrite in index)
- **Score display**: `og file#func` similar-search now shows `score: -0.423`
  instead of nonsensical `-42300% similar`
- **Boost sign bug**: `boost_results()` used `score *= boost` but omendb MaxSim
  scores are negative (less negative = more similar); fixed to `score /= boost`.
  MRR@10 improved 7.4x (0.0062 → 0.0458)
- **Help on empty args**: `og` with no arguments now prints help instead of
  exiting silently

## [0.0.1] - 2026-02-24

Initial release.
```

**Step 2: Commit**

```bash
git add CHANGELOG.md
git commit -m "Add CHANGELOG for v0.0.2"
```

### Task 5: Create GitHub Release

**Step 1: Create release from tag**

```bash
gh release create v0.0.2 \
  --title "v0.0.2" \
  --notes "$(cat <<'EOF'
## Bug Fixes

- **Scope filter**: Searching a subdirectory no longer leaks results from sibling directories with similar names
- **Markdown indexing**: Long markdown sections now correctly index all chunks (previously only the last survived)
- **Score display**: Similar search now shows raw scores instead of nonsensical negative percentages
- **Boost fix**: Core ranking bug fixed — MRR@10 improved 7.4x, first non-zero R@1 and R@5
- **UX**: `og` with no arguments now prints help
EOF
)"
```

Expected: URL to GitHub release printed.
